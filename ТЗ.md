# HotelDB — Структура базы данных

## 1. Таблица Users (Пользователи)

**Назначение:** Хранит информацию о всех пользователях системы (гости, клиенты, менеджеры).

| Поле              | Тип данных          | Ограничения                  | Описание |
|-------------------|---------------------|------------------------------|----------|
| user_id           | INT IDENTITY(1,1)   | PRIMARY KEY                  | Уникальный идентификатор пользователя |
| username          | NVARCHAR(50)        | UNIQUE NOT NULL              | Логин пользователя |
| password_hash     | NVARCHAR(255)       | NOT NULL                     | Хэш пароля |
| email             | NVARCHAR(100)       | UNIQUE NOT NULL              | Электронная почта |
| phone             | NVARCHAR(20)        | -                            | Телефон |
| full_name         | NVARCHAR(100)       | NOT NULL                     | Полное имя |
| role              | NVARCHAR(20)        | CHECK ('guest','client','manager') | Роль пользователя |
| created_at        | DATETIME2           | DEFAULT GETDATE()            | Дата создания записи |
| last_login        | DATETIME2           | NULL                         | Дата последнего входа |



## 2. Таблица Room_Types (Типы номеров)

**Назначение:** Содержит типы номеров и их базовые характеристики.

| Поле                 | Тип данных       | Ограничения      | Описание |
|----------------------|------------------|------------------|----------|
| room_type_id         | INT IDENTITY(1,1)| PRIMARY KEY      | Идентификатор типа номера |
| name                 | NVARCHAR(50)     | NOT NULL         | Название типа (Базовый, Продвинутый, Президентский) |
| description          | NVARCHAR(MAX)    | -                | Описание типа номера |
| base_price_per_night | DECIMAL(10,2)    | NOT NULL         | Базовая цена за ночь |
| max_occupancy        | INT              | NOT NULL         | Максимальное количество гостей |
| area_sqm             | INT              | NULL             | Площадь номера в м² |



## 3. Таблица Rooms (Номера)

**Назначение:** Информация о конкретных номерах в отеле.

| Поле          | Тип данных       | Ограничения                  | Описание |
|---------------|------------------|------------------------------|----------|
| room_id       | INT IDENTITY(1,1)| PRIMARY KEY                  | Идентификатор номера |
| room_number   | NVARCHAR(10)     | UNIQUE NOT NULL              | Номер комнаты (101, 202 и т.д.) |
| room_type_id  | INT              | FOREIGN KEY → Room_Types     | Тип номера |
| floor         | INT              | NOT NULL                     | Этаж |
| status        | NVARCHAR(20)     | CHECK ('active','maintenance','out_of_order') | Статус номера |
| description   | NVARCHAR(MAX)    | -                            | Дополнительное описание |



## 4. Таблица Services (Услуги)

**Назначение:** Каталог дополнительных услуг отеля.

| Поле          | Тип данных       | Ограничения                  | Описание |
|---------------|------------------|------------------------------|----------|
| service_id    | INT IDENTITY(1,1)| PRIMARY KEY                  | Идентификатор услуги |
| name          | NVARCHAR(100)    | NOT NULL                     | Название услуги |
| description   | NVARCHAR(MAX)    | -                            | Описание услуги |
| price         | DECIMAL(10,2)    | NOT NULL                     | Цена |
| unit          | NVARCHAR(30)     | CHECK (per_night, per_stay, per_person_per_night, once) | Единица измерения |
| is_active     | BIT              | DEFAULT 1                    | Активна ли услуга |



## 5. Таблица Bookings (Бронирования)

**Назначение:** Основная таблица — хранит все бронирования.

| Поле            | Тип данных       | Ограничения                  | Описание |
|-----------------|------------------|------------------------------|----------|
| booking_id      | INT IDENTITY(1,1)| PRIMARY KEY                  | Идентификатор бронирования |
| user_id         | INT              | FOREIGN KEY → Users          | Кто забронировал |
| room_id         | INT              | FOREIGN KEY → Rooms (NULL)   | Конкретный номер (может быть назначен позже) |
| room_type_id    | INT              | FOREIGN KEY → Room_Types     | Тип номера |
| check_in_date   | DATE             | NOT NULL                     | Дата заезда |
| check_out_date  | DATE             | NOT NULL                     | Дата выезда |
| num_guests      | INT              | DEFAULT 1                    | Количество гостей |
| status          | NVARCHAR(20)     | CHECK (pending, confirmed, checked_in, checked_out, cancelled, no_show) | Статус бронирования |
| total_price     | DECIMAL(12,2)    | NOT NULL                     | Общая стоимость |
| created_at      | DATETIME2        | DEFAULT GETDATE()            | Дата создания бронирования |
| updated_at      | DATETIME2        | DEFAULT GETDATE()            | Дата последнего изменения |
| created_by      | INT              | NULL                         | Кто создал (менеджер) |
| notes           | NVARCHAR(MAX)    | -                            | Примечания |



## 6. Таблица Booking_Services (Услуги в бронировании)

**Назначение:** Связующая таблица — какие услуги добавлены к конкретному бронированию.

| Поле               | Тип данных       | Ограничения                  | Описание |
|--------------------|------------------|------------------------------|----------|
| booking_service_id | INT IDENTITY(1,1)| PRIMARY KEY                  | Идентификатор записи |
| booking_id         | INT              | FOREIGN KEY → Bookings (CASCADE) | Бронирование |
| service_id         | INT              | FOREIGN KEY → Services       | Услуга |
| quantity           | INT              | DEFAULT 1                    | Количество |
| price_at_booking   | DECIMAL(10,2)    | NOT NULL                     | Цена на момент бронирования |
| subtotal           | DECIMAL(12,2)    | NOT NULL                     | Стоимость услуги |



## 7. Таблица Payments (Платежи)

**Назначение:** Хранит информацию о всех платежах по бронированиям.

| Поле            | Тип данных       | Ограничения                  | Описание |
|-----------------|------------------|------------------------------|----------|
| payment_id      | INT IDENTITY(1,1)| PRIMARY KEY                  | Идентификатор платежа |
| booking_id      | INT              | FOREIGN KEY → Bookings (CASCADE) | Бронирование |
| amount          | DECIMAL(12,2)    | NOT NULL                     | Сумма платежа |
| payment_date    | DATETIME2        | DEFAULT GETDATE()            | Дата платежа |
| payment_method  | NVARCHAR(20)     | CHECK (cash, card, online, bank_transfer) | Способ оплаты |
| status          | NVARCHAR(20)     | DEFAULT 'completed'          | Статус платежа |



## Связи между таблицами

- **Users → Bookings** (один пользователь может иметь много бронирований)
- **Room_Types → Rooms** (один тип — много номеров)
- **Room_Types → Bookings** (бронирование указывает тип номера)
- **Rooms → Bookings** (бронирование может быть привязано к конкретному номеру)
- **Bookings → Booking_Services** (одно бронирование — много услуг)
- **Services → Booking_Services** (одна услуга может быть в разных бронированиях)
- **Bookings → Payments** (одно бронирование — много платежей)
